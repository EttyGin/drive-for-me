💡 מתאים לסביבה On-Prem / Kubernetes ללא LoadBalancer.

✅ HAProxy יקבל את שם המשתמש ויפנה לפוד המתאים.
✅ כל SingleUser Notebook יקבל כתובת דינמית לפי השם שלו.

📌 הגדרת HAProxy להפניית חיבורי SSH לפי משתמש

haproxy
frontend ssh
    bind *:22
    mode tcp
    use_backend ssh_%[req.ssl_sni]

backend ssh_user1
    mode tcp
    server user1 <USER1_POD_IP>:22

backend ssh_user2
    mode tcp
    server user2 <USER2_POD_IP>:22

📌 לאחר ההגדרה, כל משתמש יוכל להתחבר ישירות דרך:

bash
ssh user1@ssh.jupyterhub.com
ssh user2@ssh.jupyterhub.com

🔹 מה הפתרון הכי טוב עבורך?

✅ אם אתה בענן עם LoadBalancer → פתרון 1 (DNS חכם)
✅ אם יש לך Ingress Controller (Nginx/Traefik) → פתרון 2 (Ingress TCP)
✅ אם אין לך LoadBalancer, רק NodePort → פתרון 3 (HAProxy)

🔹 איך גורמים ל-HAProxy להפנות אוטומטית לכל יוזר חדש?

כדי שכל SingleUser Notebook יקבל חיבור SSH בלי הגדרות ידניות, נשתמש ב-Dynamic Backend Resolution עם DNS או API של Kubernetes.

📌 שלב 1: להגדיר HAProxy עם הפנייה דינמית לפי שם המשתמש

במקום להגדיר ידנית כל יוזר, אפשר להגדיר את HAProxy כך שיפנה אוטומטית לפי הדומיין של היוזר.

haproxy
frontend ssh
    bind *:22
    mode tcp
    tcp-request inspect-delay 5s
    use_backend ssh_dynamic if { req.ssl_sni -m found }

backend ssh_dynamic
    mode tcp
    server-template user 50 _ssh._tcp.jupyterhub.svc.cluster.local resolvers kube-dns

📌 מה קורה כאן?
🔹 כל יוזר שמנסה SSH לכתובת שלו (למשל ssh user1@ssh.jupyterhub.com) → HAProxy בודק את הכתובת DNS של הפוד המתאים ומפנה אותו אוטומטית.
🔹 אין צורך להוסיף ידנית כל משתמש – HAProxy שואב את המידע אוטומטית מ-Kubernetes.

🔹 שלב 2: להשתמש בשמות DNS של הפודים כדי לפתור את הכתובת אוטומטית

ב-Kubernetes, כל SingleUser Notebook רץ בתור Pod, ואפשר לגשת אליו דרך Service שמייצר DNS אוטומטי.

לדוגמה, אם למשתמש user1 יש פוד בשם:

bash
user1-1234.ssh.jupyterhub.svc.cluster.local

אז HAProxy ידע למצוא ולהפנות אליו את החיבור בצורה אוטומטית.

אם את רוצה שההפנייה לא תהיה דרך DNS אלא דרך API של Kubernetes, אפשר להשתמש ב-HAProxy עם Lua Script שמבצע שאילתות ל-Kubernetes.

1️⃣ הסקריפט שולף רשימה של כל היוזרים (Pods)
2️⃣ HAProxy מנתב אוטומטית על סמך התוצאה

📌 דוגמה ללוגיקה:

lua
function get_pod_ip(user)
  local cmd = "kubectl get pod -l user=" .. user .. " -o jsonpath='{.items[0].status.podIP}'"
  local handle = io.popen(cmd)
  local result = handle:read("*a")
  handle:close()
  return result
end

🔹 אפשר להגדיר את HAProxy כך שיבצע קריאה ל-API של Kubernetes כל פעם שמשתמש מתחבר, ויחפש את ה-Pod שלו.

🔹 1. יצירת Service חדש לפורט 22

צרי קובץ בשם ssh-service.yaml עם התוכן הבא:

apiVersion: v1

  namespace: <הניימספייס שלך>

  selector:
    hub.jupyter.org/network-access-hub: "true"
  ports:
    - protocol: TCP
      port: 22
      targetPort: 22
  type: LoadBalancer  # אפשר גם NodePort אם LoadBalancer לא זמין

🔹 שימי לב:

  • ה-selector כאן מתייחס לפודים של Jupyter, אבל כדאי לבדוק שהטאגים תואמים את מה ש-Z2JH יצר אצלך.

  • אם את משתמשת ב-Cloud כמו GKE/AWS/EKS, LoadBalancer ייצור כתובת חיצונית לפורט 22.

  • אם את ב-On-Prem או Minikube, עדיף NodePort ואז אפשר יהיה לגשת ל-SSH דרך NodeIP:NodePort.
